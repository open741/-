# 图卡库 V2 需求说明文档

## 1. 概述
图卡库 V2 旨在为特殊教育工作者提供一个智能化、高效的视觉支持资产管理平台。平台通过整合 AI 生成、智能语义检索、以及灵活的层级文件夹管理，帮助教师快速沉淀、查找和归类教学图卡。

## 2. 智能搜索功能

图卡库提供智能搜索工作台，大幅提升教师检索教具与图卡的效率。

### 2.1 进入与退出机制
- **进入**：点击进入，页面可切换并聚焦进入智能搜索模式。
- **退出**：点击退出，可平滑退出智能搜索模式，恢复到图卡库标准视图。

### 2.2 检索机制优先级
- **优先语义搜索**：用户输入自然语言，后台通过 Embedding 向量进行语义图卡检索。
- **降级机制**：当语义检索未能搜到足够或相关度高的内容时，自动启动传统特征的**关键词匹配**。

### 2.3 辅助搜索：教案与活动关联搜索
用户可通过弹窗单选具体的【教案】或【活动】，系统会自动提取其中的内容属性转换为结构化的检索指令。只能同时添加一个教案或活动。

#### 2.3.1 教案搜索 Prompt 设定
选定教案后，系统后台自动调用大语言模型解析内容并进行查询：
```text
# Role
你是一名资深的特殊教育专家和儿童认知图卡语义解析引擎。你非常了解特殊儿童在各类教学活动中需要的视觉支持，并擅长将枯燥的教案属性转化为生动的画面描述。

# Task
你的任务是根据用户提供的【教案详情】，生成用于“向量数据库”检索的【语义描述语句】，以及用于关键词搜索的【关键词】。

# Input Format
- 教案名称：[教案名称]
- 教案活动：[活动列表（活动名称-活动步骤-训练目的）]
- 课程纲要：[课程纲要]

# Guidelines
1. 【语义检索语句 (Semantic Query)】：用自然语言描述这个教案中可能出现的所有高频视觉元素、核心动作、人物互动状态和所需物品。这句话是给机器（Embedding模型）看的，语料越丰富、画面感越强越好。
2. 【关键词 (Key words)】：提炼出3-5个最核心的名词或动词短语。用于关键词搜索匹配。

# Output Format
请严格以 JSON 格式输出，不要包含任何额外的解释文本：
{
  "semantic_query": "一段富有画面感、包含多个元素的描述性文本，用于生成向量",
  "key_words": ["关键词1", "关键词2", "关键词3"]
}
```

#### 2.3.2 活动搜索 Prompt 设定
与教案类似，选定某个活动后，进行二次扩展检索：
```text
# Role
你是一名资深的特殊教育专家和儿童认知图卡语义解析引擎。你非常了解特殊儿童在各类教学活动中需要的视觉支持，并擅长将枯燥的活动属性转化为生动的画面描述。

# Task

# Input Format
- 活动名称：[活动名称]
- 训练目的：[训练目的]
- 活动教具/学具/辅具：[教具/学具/辅具列表]

- 场景对话词：[对话词列表]

2. 【关键词 (Key words)】：提炼出3-5个最核心的名词或动词短语。用于关键词搜索匹配。

# Output Format

  "key_words": ["关键词1", "关键词2", "关键词3"]
}
```

### 2.4 搜索结果交互
- 在未分类图卡区切换显示为 **“搜索结果”** 视图。
- 显示的所有关联图卡均支持**直接鼠标拖拽**，允许老师无缝将其拉入顶部的文件夹进行归类。

## 3. 文件夹管理功能

### 3.1 视图与交互
- **展开/折叠**：文件夹大区允许自主展开或收缩以节省屏幕空间。
- **自动折叠**：当激活“智能搜索”时，文件夹区域会自动折叠以提供给搜索结果更大的屏显视图。
- **进入与退出**：
  - **进入**：点击对应文件夹卡片即可进入文件夹查看页面，顶部显示面包屑导航（如 `全部图卡 > 文件夹名` ）。
  - **退出**：点击面包屑中的 “全部图卡” 可返回主工作区。

### 3.2 内置图卡库与权限管理
- **置顶显示**：所有的“系统内置文件夹”及其内部包含的“系统内置图卡”默认处于列表最顶端。
- **保护机制**：系统内置的内容**严禁被重命名和删除**。
- **分级权限**：
  - 普通老师：**无权**向系统内置图卡库中加入新开图卡（内置系统只读）。
  - 教导主任及以上权限：拥有跨级管理能力，可以向内置图卡库中补充图卡。
  - **隔离属性**：高权用户加入的新图卡仅仅是“放置在此分类中”，该图卡的本质**不会自动升级为强制系统内置**，其仍然是属于普通用户图卡属性，可修改可编辑。

### 3.3 用户文件夹
- 用户自己创建的文件夹不再受保护机制约束。可自由**重命名**和**删除**整个目录。

---

## 4. 未分类图卡管理

所有刚生成的、或新上传暂未归档的图卡，会统一驻留在默认底部的“未分类”模块中。
- **批量操作能力**：支持单独选中、跨区多选或“一键全选 / 取消全选”。并激活浮窗底部的多功能工具栏进行：批量打印、批量下载、批量删除。
- **拖拽归类**：将单独或批量选中的图卡向文件夹进行悬停拖放，可瞬间完成分类绑定。图卡被分类后会**自动从未分类列表中消失**并进入目标文件夹。

---

## 5. 图卡操作与详情弹窗

系统内的图卡基于来源和属性会存在截然不同的悬浮菜单与详情弹窗设计。

### 5.1 用户图卡（用户上传及转移后的非系统卡）
- **操作互动**：支持进入详情页和基础交互。
- **图卡移动（右键菜单）**：在文件夹内部，对用户图卡右键唤出菜单后，可将图卡 “移动到...” 其他目录。
  - **多级菜单**：移动菜单呈现多层级结构展示系统内的文件夹树状位置。每一级菜单内部拥有“新建文件夹”入口，可直接将图卡重定向移动至现场新建的目录层级中。
  - **移除机制**：支持将其“从本文件夹移除”（放回主视图未分类）。
- **详情页体验**：使用**极简单主图弹窗模板**。
  - 对外暴露的核心功能仅限于基础的：**「下载」、「删除」**。

### 5.2 系统内置图卡
- **详情页体验**：系统卡详情页**使用**极简单主图弹窗模板**。
- **内置图卡操作**：**「下载」**

### 5.3 AI 生成的图卡
- **详情页体验**：使用专属的**生产工作台弹窗模板**。不仅有图卡预览，还会在主界面右侧常驻参数提示词解析区域。
  - **数据展示**：展示所属生成的「任务名称」、「画面描述」、生成尺寸模型比例等参数。
  - **核心操作**：除「下载」、「删除」以外，提供专有的 **「复用」** 交互。
  - **复用**：用户点击「复用」后，底层会完整复制且携带当前图片的初始生成参数设定，无缝跳转退回至 “AI生成图卡” 环节工作台内预填数据，以便进行二次修补和重新微调输出。
